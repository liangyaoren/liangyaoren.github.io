docker 中优雅关闭 tomcat 容器

发表于 2021-06-09 分类于 docker

背景

Spring Boot 项目发新版的时候，需要重启 jdk 容器。但是重启的瞬间，可能有用户请求尚未响应，如果直接停掉，会导致请求中断，用户体验不好。想了一下，有两个方案：

  • 重启 tomcat-1 节点前，用 nginx 把流量切到 tomcat-2 节点。重启 tomcat-1 节点，将流量切回到 tomcat-1 节点，继续更新 tomcat-2 节点。
  • 使用 Spring Boot 2.3 后提供的优雅关闭配置，配合 docker 容器启动命令。

显然，第二种方案更简单。
